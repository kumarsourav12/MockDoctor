# MockDoctor v0.1.0

MockDoctor compares ReadyAPI REST virtual-service files with an OpenAPI spec or a JSON contract and catches mock drift before false-green tests reach CI.

## Why this exists

Mock services are useful until they quietly stop matching reality.

That usually looks like one of these:

- the API changed but the virtual service still returns the old field
- the contract added a response path that the mock never picked up
- the response still exists, but the media type drifted
- the mock body is no longer valid JSON

The bad part is not the mismatch itself. The bad part is that tests keep passing against the wrong world.

MockDoctor reads the ReadyAPI files you commit, compares them with an OpenAPI spec or a JSON contract, and reports the exact place the drift starts.

## What shipped in v0.1.0

- ReadyAPI REST project XML input
- extracted ReadyAPI REST service-directory input
- OpenAPI contract loading and dereferencing
- JSON contract support with `operations[]` and grouped `services[]`
- drift detection for:
  - missing operations
  - missing responses
  - content-type mismatches
  - invalid JSON bodies
  - JSON schema mismatches
- text output for terminals and CI logs
- JSON output for automation
- HTML drift reports for sharing failures with teammates
- checked-in examples
- GitHub Pages presentation site

## Quick start

From this repository:

```bash
npm install
npm run build
node ./dist/cli.js compare --config ./examples/openapi/mockdoctor.config.json
```

JSON contract example:

```bash
node ./dist/cli.js compare --config ./examples/json-contract/mockdoctor.config.json
```

Schema-drift example with an HTML report:

```bash
node ./dist/cli.js compare \
  --readyapi ./test/fixtures/readyapi/orders-project-schema-drift.xml \
  --openapi ./test/fixtures/contracts/orders-openapi.yaml \
  --html-report ./artifacts/mockdoctor-drift.html
```

## Example drift

One checked-in fixture intentionally returns:

```json
{ "amount": "99" }
```

while the OpenAPI contract requires:

```yaml
amount:
  type: integer
```

MockDoctor reports:

```text
GET /api/orders/{id} | service=OrdersService
  - [response-body-schema-mismatch] ReadyAPI response OK Response for GET /api/orders/{id} does not match the contract schema.
      $.amount: must be integer
```

## Current scope

`v0.1.0` is intentionally narrow.

- ReadyAPI REST only
- OpenAPI or JSON contracts only
- non-JSON responses are compared by status code and media type only
- no dispatch-script execution or runtime virtualization
- no extra route-template inference beyond normalized literal paths
